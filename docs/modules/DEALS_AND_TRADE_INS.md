# Deals and trade-ins

The starter pack defines cash retail, third-party-financed retail, wholesale, vehicle purchase, and trade-in acquisition. Workspaces can install other deal types without code changes.

Deals link parties by roles such as buyer, seller, dealer buyer, lender, trade-in owner, and authorized representative. Inventory roles include sold, purchased, trade-in, and wholesale units.

A trade-in records vehicle facts, owner, allowance, lien/payoff, condition, odometer, title/registration details, tax-pack eligibility inputs, and resulting inventory unit. The platform never assumes a trade-in tax rule; the active tax pack evaluates it.

Official document totals reference immutable line-item, tax, and optional calculation snapshots. Closing requirements and side effects come from the configured workflow. Vynlo core contains no repossession or recurring servicing behavior.
