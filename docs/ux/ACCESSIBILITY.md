# Accessibility

Target WCAG 2.2 AA.

- Semantic landmarks/headings.
- Keyboard access and visible focus.
- Labels, descriptions, and associated errors.
- Screen-reader announcements for save/upload/generation.
- Colour is never the only status signal.
- Adequate contrast and reduced-motion support.
- Accessible reorder alternatives.
- Correct dialog/sheet focus behavior.
- Proper table headers and mobile alternatives.
- PDF templates use readable text and language metadata where practical.

Accessibility checks are part of CI and manual UAT.
